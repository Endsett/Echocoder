# UI Components

EchoCoder provides multiple UI surfaces integrated into the VS Code shell.

---

## Agent Panel

**File**: `src/ui/AgentPanelProvider.ts` | **View ID**: `echocoder.agentPanel`

The primary chat interface, rendered as a Webview in the sidebar.

### Features
- **Streaming markdown** — Agent responses render in real-time with syntax highlighting
- **Tool progress indicators** — Shows which tool the agent is currently executing
- **Token meter** — Displays input/output token count for the current session
- **User input** — Text area with send button and Enter-to-submit
- **Session state** — Tracks message history for the active session

### Usage
- Open with `Ctrl+L` or the EchoCoder icon in the Activity Bar
- Type a prompt and press Enter to start an agent run
- The agent's response streams in real-time

## Plan Viewer

**File**: `src/ui/PlanViewerProvider.ts` | **View ID**: `echocoder.planViewer`

Displays the structured plan generated by the Workflow Engine's planner phase.

### Features
- **Plan summary** with risk badge (green / yellow / red)
- **Step list** with category tags (read, write, create, execute, etc.)
- **File impact summary** — shows which files will be read and written
- **Live execution tracking** — steps update from ⏳ → 🔄 → ✅/❌ during execution
- **Approve & Execute / Reject** buttons — fixed at the bottom for easy access
- **Auto-focus** — Panel automatically opens when a plan is ready for approval

### States
| Phase | Display |
|-------|---------|
| `idle` | "No active plan" |
| `planning` | "Generating plan..." with spinner |
| `awaiting_approval` | Full plan with action buttons |
| `executing` | Steps updating in real-time |
| `completed` | All steps green |
| `failed` | Failed steps highlighted in red |

## Status Bar

**File**: `src/ui/StatusBarManager.ts`

Three status bar items provide at-a-glance state:

### Agent Status (Left side)
| State | Display |
|-------|---------|
| Idle | `$(hubot) EchoCoder` |
| Initializing | `$(sync~spin) EchoCoder: Initializing...` |
| Generating | `$(sync~spin) EchoCoder: Generating...` |
| Running tool | `$(sync~spin) EchoCoder: Running Write...` |
| Error | `$(error) EchoCoder: Error` (red background, auto-clears after 5s) |

### Model Display (Right side)
Shows the active model name (e.g., `$(symbol-misc) claude-sonnet-4-20250514`). Updates when settings change.

### Token Counter (Right side)
- Hidden until first usage event
- Shows total tokens with warning colors when approaching budget
- Click to compact context

## Session History

**File**: `src/ui/SessionHistoryProvider.ts` | **View ID**: `echocoder.sessionHistory`

A TreeView in the sidebar showing past agent sessions:

- **Chronological list** sorted newest-first
- **Summary** (first 80 characters of the session topic)
- **Timestamp** and message count
- **Persisted** to `globalState` (survives IDE restarts)
- **Limited** to last 50 sessions

## Inline Diff Decorations

**File**: `src/editor/DiffDecorator.ts`

When the agent proposes code changes, they appear as highlighted decorations directly in the editor:

| Decoration | Color | Purpose |
|-----------|-------|---------|
| Added | Green background `rgba(40,167,69,0.15)` | New lines to be inserted |
| Removed | Red background `rgba(220,53,69,0.1)` | Lines to be deleted |
| Modified | Yellow left border `rgba(255,193,7,0.6)` | Lines being changed |

Hovering over a decorated region shows a markdown diff popup with accept/reject instructions.

## CodeLens Approval

**File**: `src/editor/CodeLensApprovalProvider.ts`

Renders clickable buttons directly above each pending change:

```
✅ Accept  |  ❌ Reject  |  ✅ Accept All
```

- Only appears when `echocoder.hasPendingChanges` is `true`
- "Accept All" only shows when multiple changes are pending
- Updates dynamically as changes are accepted/rejected
